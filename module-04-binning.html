MODULE 4 — Day 3, Morning

Contig Binning

Group assembled contigs into draft genomes (bins / MAGs) using coverage depth and tetranucleotide composition with six complementary binners and Tiara domain classification.

Learning Objectives

  • Understand the two core binning signals: coverage depth and tetranucleotide composition
  • Prepare coverage depth files from BAM alignments for binners
  • Run MetaBAT2, MaxBin2, CONCOCT, COMEBin, MetaBinner, and SemiBin2
  • Classify contigs by domain (Bacteria, Archaea, Eukaryota, Virus) with Tiara before binning
  • Compare bin sets from different binners to inform DAS Tool refinement

Background

Binning clusters assembled contigs into draft genomes by exploiting two signals: (1) coverage depth — contigs from the same organism have similar depth across samples; (2) tetranucleotide frequencies — species have characteristic k-mer composition (genomic signatures). Different binners weight these signals differently. Running multiple binners and combining them with DAS Tool (Module 5) consistently outperforms any single binner. Tiara classifies contigs by domain using deep learning on k-mer features, allowing eukaryotic, viral, and prokaryotic contigs to be handled separately before or during binning.

Step 1 — Prepare Coverage Depth File

Most binners require a depth file derived from the BAM alignment generated in Module 2:

mkdir -p results/04-binning/coverage

# Generate MetaBAT2-compatible depth file
jgi_summarize_bam_contig_depths \
    --outputDepth results/04-binning/coverage/sample.depth.txt \
    results/02-assembly/coverage/sample.sorted.bam

Step 2 — Domain Classification with Tiara

mkdir -p results/04-binning/tiara

tiara \
    -i results/02-assembly/megahit/sample.contigs.fa \
    -o results/04-binning/tiara/sample.tiara.txt \
    --threads 8 \
    --probabilities

# View domain assignment summary
cut -f2 results/04-binning/tiara/sample.tiara.txt | sort | uniq -c
Tiara output Tiara assigns each contig to: bacteria, archaea, eukarya, organelle, virus, or unknown. In nf-core/mag, Tiara is used to separate eukaryotic contigs for MetaEuk annotation and to refine binning of prokaryotic contigs.

Step 3 — MetaBAT2

mkdir -p results/04-binning/metabat2

metabat2 \
    -i results/02-assembly/megahit/sample.contigs.fa \
    -a results/04-binning/coverage/sample.depth.txt \
    -o results/04-binning/metabat2/bin \
    --minContig 2000 \
    --numThreads 8 \
    --saveCls

Step 4 — MaxBin2

mkdir -p results/04-binning/maxbin2

run_MaxBin.pl \
    -contig results/02-assembly/megahit/sample.contigs.fa \
    -abund results/04-binning/coverage/sample.depth.txt \
    -out results/04-binning/maxbin2/bin \
    -min_contig_length 2000 \
    -thread 8

Step 5 — CONCOCT

mkdir -p results/04-binning/concoct/cut_contigs
mkdir -p results/04-binning/concoct/output

# Cut contigs into chunks for CONCOCT
cut_up_fasta.py \
    results/02-assembly/megahit/sample.contigs.fa \
    -c 10000 -o 0 --merge_last \
    -b results/04-binning/concoct/cut_contigs/sample.bed \
    > results/04-binning/concoct/cut_contigs/sample.cut.fa

# Generate coverage table
concoct_coverage_table.py \
    results/04-binning/concoct/cut_contigs/sample.bed \
    results/02-assembly/coverage/sample.sorted.bam \
    > results/04-binning/concoct/output/coverage_table.tsv

# Run CONCOCT
concoct \
    --composition_file results/04-binning/concoct/cut_contigs/sample.cut.fa \
    --coverage_file results/04-binning/concoct/output/coverage_table.tsv \
    -b results/04-binning/concoct/output/ \
    --threads 8

# Merge cut contigs back
merge_cutup_clustering.py \
    results/04-binning/concoct/output/clustering_gt1000.csv \
    > results/04-binning/concoct/output/clustering_merged.csv

# Extract bins
extract_fasta_bins.py \
    results/02-assembly/megahit/sample.contigs.fa \
    results/04-binning/concoct/output/clustering_merged.csv \
    --output_path results/04-binning/concoct/bins/

Step 6 — COMEBin

mkdir -p results/04-binning/comebin

run_comebin.sh \
    -a results/02-assembly/megahit/sample.contigs.fa \
    -o results/04-binning/comebin \
    -p results/02-assembly/coverage/ \
    -t 8

Step 7 — SemiBin2

mkdir -p results/04-binning/semibin2

# Single-sample mode with self-supervised training
SemiBin2 single_easy_bin \
    -i results/02-assembly/megahit/sample.contigs.fa \
    -b results/02-assembly/coverage/sample.sorted.bam \
    -o results/04-binning/semibin2 \
    --environment human_gut \
    --threads 8
SemiBin2 environment presets Use --environment to select a pretrained model: human_gut, dog_gut, ocean, soil, built_environment, wastewater, cat_gut, global. Use global when no specific preset fits.

Step 8 — Count Bins per Binner

# Count bins from each binner
echo "MetaBAT2:  $(ls results/04-binning/metabat2/bin.*.fa 2>/dev/null | wc -l) bins"
echo "MaxBin2:   $(ls results/04-binning/maxbin2/bin.*.fasta 2>/dev/null | wc -l) bins"
echo "SemiBin2:  $(ls results/04-binning/semibin2/output_bins/*.fa 2>/dev/null | wc -l) bins"
