DAY 1, 09:30 – 10:00

Nextflow & nf-core/mag Overview

Understand how Nextflow works, explore the nf-core/mag parameter space, build a samplesheet, and submit the pipeline job that will run in the background while you work through the manual modules.

Learning Objectives

  • Understand the relationship between Nextflow, nf-core, and nf-core/mag
  • Load Nextflow on Ibex and verify the installation
  • Navigate the nf-core/mag parameter space with --help
  • Build a valid samplesheet CSV for your dataset
  • Validate the pipeline configuration with a test run (-profile test)
  • Submit the full pipeline job to run overnight

Background

Nextflow is a workflow manager that chains bioinformatics tools into reproducible, scalable pipelines. It handles parallelisation, error recovery, and execution on HPC clusters automatically. nf-core is a community effort that provides curated, best-practice Nextflow pipelines, including nf-core/mag, which automates every step you will run manually in Modules 1–8. Running the pipeline in parallel with the manual sessions lets you compare automated and manual outputs at the end of the course (Day 4).

Step 1: Load Nextflow on Ibex

# create a workdir for nf-core/mag workflow
cd /ibex/user/$USER/metagenomics_course
mkdir -p nfcore_mag
cd nfcore_mag;

# Load the Nextflow & singularity
module load nextflow singularity

# Verify
nextflow -version

Expected output:

N E X T F L O W
version XX.xx.xx build xxxx
...
Singularity on Ibex nf-core/mag uses Singularity containers to run every tool: no conda environments needed when running through the pipeline. Singularity is pre-installed on all Ibex compute nodes. We use -profile singularity in all pipeline commands. However, the orgnisational profile kaust is set to use singularity by default.

Step 3: Build the Samplesheet

nf-core/mag takes a CSV samplesheet as input. Each row is one sample:

ColumnRequiredDescription
sampleYesUnique sample name (no spaces)
runnowhen a run column is present, the pipeline will also perform run- or lane-wise concatenation, for cases where you may have a sample or library sequenced with the same sequencing configuration across multiple runs.
groupYesSamples sharing a group are co-assembled and use each other for binning coverage
short_reads_1ConditionalPath to forward FASTQ (can be gzipped). Remove if long reads only.
short_reads_2ConditionalPath to reverse FASTQ. Remove if long reads only or single reads.
long_readsConditionalPath to long-read FASTQ: leave empty for short-read-only samples
short_reads_platformConditionalRequired if short reads are given. Values: ILLUMINA, BGISEQ, LS454, ION_TORRENT, DNBSEQ, ELEMENT, ULTIMA, VELA_DIAGNOSTICS, GENAPSYS, GENEMIND, TAPESTRI
long_reads_platformConditionalRequired if long reads are given. Values: OXFORD_NANOPORE, OXFORD_NANOPORE_HQ, PACBIO_CLR, PACBIO_HIFI
 
# Create the samplesheet
cat > samplesheet.csv << 'EOF'
sample,group,short_reads_1,short_reads_2,long_reads,short_reads_platform,long_reads_platform
mock_control,1,/ibex/user/$USER/metagenomics_course/raw_reads/mock_control_1.fastq.gz,/ibex/user/$USER/metagenomics_course/raw_reads/mock_control_2.fastq.gz,,ILLUMINA
mock_ss90,1,/ibex/user/$USER/metagenomics_course/raw_reads/mock_ss90_1.fastq.gz,/ibex/user/$USER/metagenomics_course/raw_reads/mock_ss90_2.fastq.gz,,ILLUMINA
mock_ont,2,,,/ibex/user/$USER/metagenomics_course/raw_reads/zymo-even_ont-r10_long.fastq.gz,,OXFORD_NANOPORE_HQ
EOF

# Verify it looks correct
cat samplesheet.csv
Indepth info regarding input and samplesheet format can be found on the nf-core/mag usage page.
Groups and co-assembly Samples in the same group are co-assembled by default when --coassemble_group true is set. For independent assemblies, give each sample a unique group name. Coverage from all group members is used to improve binning regardless of assembly mode.

Step 4: Validate with a Test Run

The -profile ,test Test profile uses small datasets that complete quickly while exercising all pipeline features.

# allocate an interactive session
srun -t 4:00:00 -c 15 --pty bash

# run a test set
nextflow run nf-core/mag \
    -r 5.4.2 \
    -profile kaust,test \
    --outdir results/test

A successful test run using a small test dataset.

Step 5: Submit the Pipeline Job

Once the test run finishes, submit the real job as a SLURM batch script. This will run in the background while you work through the manual modules.

cat > run_nfcore_mag.sh << 'EOF'
#!/bin/bash
#SBATCH --job-name=nfcore_mag_mock
#SBATCH --partition=batch
#SBATCH --time=48:00:00
#SBATCH --mem=8G
#SBATCH --cpus-per-task=4
#SBATCH --output=nfcore_mag_%j.log
#SBATCH --error=nfcore_mag_%j.err

module load nextflow

nextflow run nf-core/mag -resume \
    -r 5.4.2 \
    -profile kaust \
    --input samplesheet.csv \
    --outdir results/ \
    --host_fasta /biocorelab/BIX/resources/genomes/mus_musculus/release-113/Mus_musculus.GRCm39.dna_sm.toplevel.fa \
    --skip_spades false \
    --run_virus_identification true \
    --genomad_db /ibex/reference/KSL/genomad/v1.5/genomad_db \
    --skip_gunc false --gunc_save_db true \
    --checkm2_db /ibex/sw/rl9c/checkm2/1.0.2/rl9_conda3/database/CheckM2_database/uniref100.KO.1.dmnd \
    --gtdb_db /ibex/reference/KSL/gtdb-tk/release226/ \
    --coassemble_group false --prokka_compliance_centre \
    --refine_bins_dastool true --postbinning_input both \
    --generate_bigmag_file true \
    --run_checkm true --run_checkm2 --run_quast
EOF

# Submit
sbatch run_nfcore_mag.sh

# Check it is queued or running
squeue -u $USER
Nextflow resumes from where it stopped If the job fails or times out, you can re-submit with -resume added to the nextflow run command. Nextflow caches completed process outputs and only re-runs failed or missing steps.
