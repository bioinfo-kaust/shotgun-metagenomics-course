MODULE 4, DAY 3, 09:15 – 12:00

Contig Binning

Group assembled contigs into draft genomes (bins / MAGs) using coverage depth and tetranucleotide composition with six complementary binners and Tiara domain classification.

Learning Objectives

  • Understand the two core binning signals: coverage depth and tetranucleotide composition
  • Prepare coverage depth files from BAM alignments for binners
  • Run MetaBAT2, MaxBin2, CONCOCT, COMEBin, and SemiBin2
  • Classify contigs by domain (Bacteria, Archaea, Eukaryota, Virus) with Tiara before binning
  • Compare bin sets from different binners to inform DAS Tool refinement

Background

Binning clusters assembled contigs into draft genomes by exploiting two signals: (1) coverage depth: contigs from the same organism have similar depth across samples; (2) tetranucleotide frequencies: species have characteristic k-mer composition (genomic signatures). Different binners weight these signals differently. Running multiple binners and combining them with DAS Tool (Module 5) consistently outperforms any single binner. Tiara classifies contigs by domain using deep learning on k-mer features, allowing eukaryotic, viral, and prokaryotic contigs to be handled separately before or during binning.

Tools

ToolStrategynf-core/mag parameter
MetaBAT2Coverage + composition; fast & robust--skip_metabat2 false
MaxBin2EM clustering on coverage + 107 universal marker genes--skip_maxbin2 false
CONCOCTGaussian mixture model on coverage + k-mers--skip_concoct false
COMEBinContrastive learning on coverage + composition--skip_comebin false
SemiBin2Self-supervised deep learning; sample or multi-sample mode--skip_semibin false
TiaraDeep-learning domain classification of contigs--skip_tiara false

Step 1: Prepare Coverage Depth File

Most binners require a depth file derived from the BAM alignment generated in Module 2:

cat > scripts/bin_01_coverage_depth.sh << 'EOF'
#!/bin/bash
#SBATCH --job-name=bin_01_coverage_depth
#SBATCH --partition=batch
#SBATCH --time=00:30:00
#SBATCH --mem=64G
#SBATCH --cpus-per-task=4
#SBATCH --output=logs/bin_01_coverage_depth_%j.out
#SBATCH --error=logs/bin_01_coverage_depth_%j.err
#SBATCH --mail-type=END,FAIL
#SBATCH --mail-user=

# Exit immediately on error (-e), treat unset variables as errors (-u),
# and propagate pipe failures (-o pipefail) so the job stops at the first
# sign of trouble rather than silently continuing with bad data.
set -euo pipefail
cd /ibex/user/$USER/metagenomics_course
mkdir -p results/04-binning/coverage

module load metabat

jgi_summarize_bam_contig_depths \
    --outputDepth results/04-binning/coverage/mock_control.depth.txt \
    results/02-assembly/coverage/mock_control.sorted.bam
EOF

Submit: sbatch scripts/bin_01_coverage_depth.sh  |  Monitor: squeue -u $USER

Step 2: Domain Classification with Tiara

cat > scripts/bin_02_tiara.sh << 'EOF'
#!/bin/bash
#SBATCH --job-name=bin_02_tiara
#SBATCH --partition=batch
#SBATCH --time=00:30:00
#SBATCH --mem=64G
#SBATCH --cpus-per-task=16
#SBATCH --output=logs/bin_02_tiara_%j.out
#SBATCH --error=logs/bin_02_tiara_%j.err
#SBATCH --mail-type=END,FAIL
#SBATCH --mail-user=
#SBATCH --export=NONE

# Exit immediately on error (-e), treat unset variables as errors (-u),
# and propagate pipe failures (-o pipefail) so the job stops at the first
# sign of trouble rather than silently continuing with bad data.
set -euo pipefail
cd /ibex/user/$USER/metagenomics_course
mkdir -p results/04-binning/tiara

source $(conda info --base)/etc/profile.d/conda.sh 2>/dev/null ||\
    source /ibex/user/$USER/miniforge3/etc/profile.d/conda.sh
conda activate tiara

tiara \
    -i results/02-assembly/megahit/mock_control.contigs.fa \
    -o results/04-binning/tiara/mock_control.tiara.txt \
    --threads 16 \
    --probabilities
EOF

Submit: sbatch scripts/bin_02_tiara.sh  |  Monitor: squeue -u $USER

Once the job completes, review the domain assignment summary interactively:

cut -f2 results/04-binning/tiara/mock_control.tiara.txt | sort | uniq -c
Tiara output Tiara assigns each contig to: bacteria, archaea, eukarya, organelle, virus, or unknown. In nf-core/mag, Tiara is used to separate eukaryotic contigs for MetaEuk annotation and to refine binning of prokaryotic contigs.

Step 5: CONCOCT

cat > scripts/bin_05_concoct.sh << 'EOF'
#!/bin/bash
#SBATCH --job-name=bin_05_concoct
#SBATCH --partition=batch
#SBATCH --time=02:00:00
#SBATCH --mem=32G
#SBATCH --cpus-per-task=8
#SBATCH --output=logs/bin_05_concoct_%j.out
#SBATCH --error=logs/bin_05_concoct_%j.err
#SBATCH --mail-type=END,FAIL
#SBATCH --mail-user=

# Exit immediately on error (-e), treat unset variables as errors (-u),
# and propagate pipe failures (-o pipefail) so the job stops at the first
# sign of trouble rather than silently continuing with bad data.
set -euo pipefail
cd /ibex/user/$USER/metagenomics_course
mkdir -p results/04-binning/concoct/cut_contigs
mkdir -p results/04-binning/concoct/output
mkdir -p results/04-binning/concoct/bins

module load concoct

cut_up_fasta.py \
    results/02-assembly/megahit/mock_control.contigs.fa \
    -c 10000 -o 0 --merge_last \
    -b results/04-binning/concoct/cut_contigs/mock_control.bed \
    > results/04-binning/concoct/cut_contigs/mock_control.cut.fa

concoct_coverage_table.py \
    results/04-binning/concoct/cut_contigs/mock_control.bed \
    results/02-assembly/coverage/mock_control.sorted.bam \
    > results/04-binning/concoct/output/coverage_table.tsv

concoct \
    --composition_file results/04-binning/concoct/cut_contigs/mock_control.cut.fa \
    --coverage_file results/04-binning/concoct/output/coverage_table.tsv \
    -b results/04-binning/concoct/output/ \
    --threads 8

merge_cutup_clustering.py \
    results/04-binning/concoct/output/clustering_gt1000.csv \
    > results/04-binning/concoct/output/clustering_merged.csv

extract_fasta_bins.py \
    results/02-assembly/megahit/mock_control.contigs.fa \
    results/04-binning/concoct/output/clustering_merged.csv \
    --output_path results/04-binning/concoct/bins/
EOF

Submit: sbatch scripts/bin_05_concoct.sh  |  Monitor: squeue -u $USER
