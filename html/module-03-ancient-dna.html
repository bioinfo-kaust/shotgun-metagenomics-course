MODULE 3, DAY 2, 14:40 – 15:15

Ancient DNA Analysis

Identify and quantify DNA damage patterns characteristic of ancient samples, call consensus variants, and correct assemblies for deamination damage using PyDamage, FreeBayes, and BCFtools.

Learning Objectives

  • Understand patterns of post-mortem DNA damage (C→T deamination at 5' ends, G→A at 3' ends)
  • Quantify per-contig damage rates with PyDamage to confirm ancient origin
  • Call high-confidence variants with FreeBayes to distinguish true variants from damage artefacts
  • Generate damage-corrected consensus sequences with BCFtools
  • Understand how nf-core/mag integrates the ancient DNA validation workflow

Background

Ancient DNA (aDNA) is characterised by hydrolytic deamination of cytosine to uracil, which is read as thymine during sequencing. This produces a C→T excess at the 5' end of reads and a complementary G→A excess at the 3' end: the diagnostic damage pattern. PyDamage models this damage per contig, allowing contigs with significant aDNA signal to be identified. Once damage-affected contigs are confirmed, FreeBayes calls consensus variants, and BCFtools applies them to produce a corrected FASTA. This workflow is optional in nf-core/mag and is activated for samples where ancient origin is expected.

Tools

ToolPurposenf-core/mag parameter
PyDamageQuantify aDNA damage per contig; filter for ancient origin--ancient_dna true
FreeBayesVariant calling for consensus correction(part of ancient DNA mode)
BCFtoolsApply variants to produce corrected FASTA(part of ancient DNA mode)

Hands-on

Step 1: Map Reads to Assembled Contigs

Ancient DNA analysis requires an alignment BAM file. Map the QC-filtered reads to the MEGAHIT assembly:

cat > scripts/adna_01_mapping.sh << 'EOF'
#!/bin/bash
#SBATCH --job-name=adna_01_mapping
#SBATCH --partition=batch
#SBATCH --time=02:00:00
#SBATCH --mem=16G
#SBATCH --cpus-per-task=8
#SBATCH --output=logs/adna_01_mapping_%j.out
#SBATCH --error=logs/adna_01_mapping_%j.err
#SBATCH --mail-type=END,FAIL
#SBATCH --mail-user=

# Exit immediately on error (-e), treat unset variables as errors (-u),
# and propagate pipe failures (-o pipefail) so the job stops at the first
# sign of trouble rather than silently continuing with bad data.
set -euo pipefail
cd /ibex/user/$USER/metagenomics_course
mkdir -p results/03-ancient-dna/mapping

module load bowtie2 samtools

bowtie2-build \
    results/02-assembly/megahit/mock_control.contigs.fa \
    results/03-ancient-dna/mapping/assembly_idx

bowtie2 \
    -x results/03-ancient-dna/mapping/assembly_idx \
    -1 results/01-qc/bbnorm/mock_control_1.norm.fastq.gz \
    -2 results/01-qc/bbnorm/mock_control_2.norm.fastq.gz \
    -p 8 | \
samtools sort -o results/03-ancient-dna/mapping/mock_control.sorted.bam
samtools index results/03-ancient-dna/mapping/mock_control.sorted.bam
EOF

Submit: sbatch scripts/adna_01_mapping.sh  |  Monitor: squeue -u $USER

Step 2: Quantify DNA Damage with PyDamage

cat > scripts/adna_02_pydamage.sh << 'EOF'
#!/bin/bash
#SBATCH --job-name=adna_02_pydamage
#SBATCH --partition=batch
#SBATCH --time=01:00:00
#SBATCH --mem=16G
#SBATCH --cpus-per-task=8
#SBATCH --output=logs/adna_02_pydamage_%j.out
#SBATCH --error=logs/adna_02_pydamage_%j.err
#SBATCH --mail-type=END,FAIL
#SBATCH --mail-user=

# Exit immediately on error (-e), treat unset variables as errors (-u),
# and propagate pipe failures (-o pipefail) so the job stops at the first
# sign of trouble rather than silently continuing with bad data.
set -euo pipefail
cd /ibex/user/$USER/metagenomics_course
mkdir -p results/03-ancient-dna/pydamage

source $(conda info --base)/etc/profile.d/conda.sh 2>/dev/null ||\
    source /ibex/user/$USER/miniforge3/etc/profile.d/conda.sh
conda activate metagenomics

pydamage analyze \
    results/03-ancient-dna/mapping/mock_control.sorted.bam \
    --outdir results/03-ancient-dna/pydamage \
    --process 8 \
    --min_cov 5

pydamage filter \
    results/03-ancient-dna/pydamage/pydamage_results.csv
EOF

Submit: sbatch scripts/adna_02_pydamage.sh  |  Monitor: squeue -u $USER

Interpreting PyDamage output Key columns in pydamage_results.csv: damage_model_p_value (significant damage if <0.05), damage_model_freq (C→T rate at 5' position 1), predicted_accuracy. A high damage_model_freq (>10%) with low p-value indicates genuine ancient origin.

Step 3: Variant Calling with FreeBayes

FreeBayes calls SNPs across damage-positive contigs to distinguish true variants from deamination artefacts:

cat > scripts/adna_03_freebayes.sh << 'EOF'
#!/bin/bash
#SBATCH --job-name=adna_03_freebayes
#SBATCH --partition=batch
#SBATCH --time=02:00:00
#SBATCH --mem=16G
#SBATCH --cpus-per-task=8
#SBATCH --output=logs/adna_03_freebayes_%j.out
#SBATCH --error=logs/adna_03_freebayes_%j.err
#SBATCH --mail-type=END,FAIL
#SBATCH --mail-user=

# Exit immediately on error (-e), treat unset variables as errors (-u),
# and propagate pipe failures (-o pipefail) so the job stops at the first
# sign of trouble rather than silently continuing with bad data.
set -euo pipefail
cd /ibex/user/$USER/metagenomics_course
mkdir -p results/03-ancient-dna/freebayes

module load freebayes

freebayes \
    -f results/02-assembly/megahit/mock_control.contigs.fa \
    results/03-ancient-dna/mapping/mock_control.sorted.bam \
    --min-coverage 3 \
    --min-alternate-fraction 0.33 \
    --pooled-continuous \
    > results/03-ancient-dna/freebayes/mock_control.vcf
EOF

Submit: sbatch scripts/adna_03_freebayes.sh  |  Monitor: squeue -u $USER

Step 4: Consensus Correction with BCFtools

These file operations complete in seconds; run them interactively once Step 3 finishes.

mkdir -p results/03-ancient-dna/corrected

module load bcftools

# Compress and index VCF
bgzip results/03-ancient-dna/freebayes/mock_control.vcf
tabix -p vcf results/03-ancient-dna/freebayes/mock_control.vcf.gz

# Apply variants to generate corrected consensus
bcftools consensus \
    -f results/02-assembly/megahit/mock_control.contigs.fa \
    results/03-ancient-dna/freebayes/mock_control.vcf.gz \
    > results/03-ancient-dna/corrected/mock_control.corrected.fa

Step 5: Inspect Damage Patterns

# View damage summary for top contigs
head -20 results/03-ancient-dna/pydamage/pydamage_results.csv | \
    cut -d',' -f1,2,3,4,5 | column -t -s','

# Count contigs with significant damage (q-value < 0.05)
awk -F',' 'NR>1 && $4 < 0.05 {count++} END {print count " contigs with significant damage"}' \
    results/03-ancient-dna/pydamage/pydamage_results.csv
