Environment Setup

Complete these steps before the first day of the course. All sessions run on the KAUST Ibex HPC cluster.

Step 1 — Ibex Account & SSH Access

All course sessions run on the Ibex HPC cluster. You need a KAUST Ibex account before Day 1.

  • KAUST staff and students: request access via the my.ibex portal
  • External collaborators: contact your KAUST PI — external accounts require PI sponsorship and VPN access with destination ibex-login

Verify you can connect before the course:

ssh -X <username>@ilogin.ibex.kaust.edu.sa

See the Linux & Ibex lab for a full walkthrough of the cluster environment.

Login nodes are for job submission only Do not run tools or scripts directly on the login node. All hands-on steps use srun (interactive) or sbatch (batch) to run on compute nodes.

Step 2 — Set Up Your Working Directory

Use /ibex/user/$USER for all course files. It provides 1.5 TB of high-performance WekaIO storage and persists for the lifetime of your account — do not use your home directory for large data.

# Create your course working directory
mkdir -p /ibex/user/$USER/metagenomics_course
cd /ibex/user/$USER/metagenomics_course

# Add WORKDIR to your shell profile for convenience
echo 'export WORKDIR=/ibex/user/$USER/metagenomics_course' >> ~/.bashrc
source ~/.bashrc

echo $WORKDIR

Step 3 — Request Reference Database Access

Many bioinformatics databases are pre-installed on Ibex. Access requires a one-time request via the my.ibex portal — approval is automatic.

  1. Go to my.ibex.kaust.edu.sa/v2/references

Step 4 — Set Up Conda Environments

Ibex provides Miniforge via the module system — no separate installation needed. Load it and create the course environments:

# Load Miniforge (provides mamba and conda)
module load miniforge3

# Initialise conda in your shell (only needed once)
conda init bash
source ~/.bashrc

Create one environment per module group:

# QC & Preprocessing (Module 1)
mamba create -n metagenomics-qc -c conda-forge -c bioconda -y \
    fastqc fastp bowtie2 samtools bbmap nanoplot porechop chopper filtlong nanoq

# Assembly & Annotation (Module 2)
mamba create -n metagenomics-assembly -c conda-forge -c bioconda -y \
    megahit spades flye quast prodigal genomad minimap2 samtools ale

# Ancient DNA (Module 3)
mamba create -n metagenomics-ancientdna -c conda-forge -c bioconda -y \
    pydamage freebayes bcftools samtools

# Binning (Modules 4–5)
mamba create -n metagenomics-binning -c conda-forge -c bioconda -y \
    metabat2 maxbin2 concoct comebin semibin2 tiara das_tool

# Bin QC (Module 6)
mamba create -n metagenomics-binqc -c conda-forge -c bioconda -y \
    checkm2 busco gunc quast

# Taxonomy & Annotation (Module 7)
mamba create -n metagenomics-taxonomy -c conda-forge -c bioconda -y \
    gtdbtk cat prokka metaeuk

# Summary (Module 8)
mamba create -n metagenomics-summary -c conda-forge -c bioconda -y \
    multiqc bigmag
Why separate environments? Many metagenomics tools have conflicting dependencies. Separate environments per module group avoids conflicts and makes it easier to troubleshoot installation issues.

Step 5 — Install Nextflow

Nextflow is available as an Ibex module — no installation required:

module load nextflow/23.10.0
nextflow -version

To make it available in every session, add it to your profile:

echo 'module load nextflow/23.10.0' >> ~/.bashrc

Step 6 — Verify Your Setup

Run these checks on a compute node (via srun) to confirm everything is ready:

# Start an interactive compute session first
srun --partition=batch --time=00:30:00 --mem=4G --cpus-per-task=2 --pty bash

# Check working directory
echo $WORKDIR
df -h /ibex/user/$USER

# Check conda environments
conda env list

# Spot-check key tools
conda activate metagenomics-qc  && fastqc --version && fastp --version && conda deactivate
conda activate metagenomics-assembly && megahit --version && conda deactivate
conda activate metagenomics-binning  && metabat2 --help 2>&1 | head -2 && conda deactivate
conda activate metagenomics-binqc    && checkm2 --version && conda deactivate
conda activate metagenomics-taxonomy && gtdbtk --version && conda deactivate

# Check Nextflow
module load nextflow/23.10.0 && nextflow -version
All commands should print a version number without errors. If an environment fails to activate, check that you ran conda init bash && source ~/.bashrc in Step 4. If a tool is missing, re-run the mamba create command for that environment.
