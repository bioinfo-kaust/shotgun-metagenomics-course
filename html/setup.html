Environment Setup

Complete these steps before the first day of the course. All sessions run on the KAUST Ibex HPC cluster.

Step 1: Ibex Account & SSH Access

All course sessions run on the Ibex HPC cluster. You need an active Ibex account associated with a PI before Day 1.

1a: Create your Ibex account

  • KAUST staff and students: request access via the my.ibex portal
  • External collaborators: contact your KAUST PI: external accounts require PI sponsorship and VPN access
Associate your account with a PI: required before logging in Without PI association your account cannot submit jobs.
  1. Log in to my.ibex.kaust.edu.sa
  2. Go to Membership Management
  3. Search for your PI and click Request Membership
  4. Your PI will receive an email with an approval link: ask them to approve it promptly

1b: Connect via SSH

macOS: Open the built-in Terminal (Applications → Utilities → Terminal) and run:

ssh <username>@ilogin.ibex.kaust.edu.sa

Windows: Download and install MobaXterm Home Edition (free). Then:

  1. Launch MobaXterm and click Session → SSH
  2. Set Remote host to ilogin.ibex.kaust.edu.sa
  3. Tick Specify username and enter your Ibex username
  4. Click OK: MobaXterm will prompt for your password

Alternatively, use the MobaXterm built-in terminal directly:

ssh <username>@ilogin.ibex.kaust.edu.sa
Login nodes are for job submission only Do not run tools or analyses directly on the login node. All hands-on steps use srun (interactive) or sbatch (batch) to run on compute nodes.

See the Linux & Ibex lab for a full walkthrough of the cluster environment.

Step 2: Set Up Your Working Directory

Use /ibex/user/$USER for all course files. It provides 1.5 TB of high-performance WekaIO storage and persists for the lifetime of your account. Do not use your home directory for large data.

# Create your course working directory
mkdir -p /ibex/user/$USER/metagenomics_course

# Navigate into it
cd /ibex/user/$USER/metagenomics_course

# Confirm available space
df -h /ibex/user/$USER
At the start of each module you will cd into this directory. No environment variables are set; just use the full path or navigate manually each session.

Step 3: Set Up the Course Conda Environment

Most bioinformatics tools used in this course are pre-installed on Ibex and loaded with module load. For any tool not available as an Ibex module, we use a single shared conda environment called metagenomics.

Install Miniforge3 directly into your user space on Ibex (run once):

cd /ibex/user/$USER

# Download Miniforge3 installer
wget https://github.com/conda-forge/miniforge/releases/latest/download/Miniforge3-Linux-x86_64.sh

# Install silently into your user space
bash Miniforge3-Linux-x86_64.sh -b -p /ibex/user/$USER/miniforge3

# Remove the installer
rm Miniforge3-Linux-x86_64.sh

# Activate conda for the current session (no .bashrc modification)
source /ibex/user/$USER/miniforge3/etc/profile.d/conda.sh
Run the source line above at the start of any session where you need conda. It activates conda for that session only, without modifying your shell profile. The sbatch scripts for tools not available as Ibex modules use a self-sourcing guard that works the same way: 
source $(conda info --base)/etc/profile.d/conda.sh 2>/dev/null || \
    source /ibex/user/$USER/miniforge3/etc/profile.d/conda.sh

Create the course environments

Three conda environments are needed. Run these on a compute node; creating all three takes 10–15 minutes:

srun --partition=batch --time=01:00:00 --cpus-per-task=4 --pty bash

# ── 1. Main environment (Python 3.10 avoids version conflicts) ───────────────
conda create -n metagenomics python=3.10 -c conda-forge -c bioconda -y
conda activate metagenomics
mamba install -c conda-forge -c bioconda -y \
  pydamage \
  gunc \
  semibin \
  metaeuk \
  ale \
  metamdbg \
  chopper \
  nanoq \
  bigmag \
  porechop_abi
conda deactivate

# ── 2. Tiara environment ──────────────────────────────────────────────────────
# Tiara requires joblib <0.18, which conflicts with COMEBin (joblib =1.1.0),
# so it lives in its own environment.
conda create -n tiara python=3.9 -c conda-forge -c bioconda tiara -y

# ── 3. COMEBin environment ────────────────────────────────────────────────────
conda create -n comebin -c conda-forge -c bioconda comebin -y

# ── Verify ────────────────────────────────────────────────────────────────────
conda activate metagenomics
pydamage --version && gunc --version && SemiBin2 --version && \
metaeuk version && chopper --version && nanoq --version && \
bigmag --version && porechop_abi --version
conda deactivate
conda run -n tiara tiara --version
conda run -n comebin run_comebin.sh --help 2>&1 | head -1
Three environments, three purposes Most tools that are not available as Ibex modules use conda activate metagenomics. Tiara uses conda activate tiara and COMEBin uses conda activate comebin. The sbatch scripts for each tool activate the correct environment automatically.

Step 4: Load Nextflow & Singularity

Nextflow and Singularity are both pre-installed on Ibex. Always load them together: Singularity is required for container-based pipeline execution:

module load nextflow singularity

# Verify both are available
nextflow -version
singularity --version
Use -profile kaust All nf-core/mag runs in this course use the KAUST Ibex profile, which configures Slurm resources and Singularity automatically:
nextflow run nf-core/mag -profile kaust ...
Details about the KAUST profile for NextFlow, the nf-core/mag workflow can be found at https://nf-co.re/configs/kaust/

Step 5: Verify Your Setup

Run these checks on a compute node to confirm everything is ready:

# Start an interactive compute session
srun --partition=batch --time=00:30:00 --mem=4G --cpus-per-task=2 --pty bash

# Check your working directory exists and has space
cd /ibex/user/$USER/metagenomics_course
df -h /ibex/user/$USER

# Check the conda environment (activate first)
source $(conda info --base)/etc/profile.d/conda.sh 2>/dev/null || \
    source /ibex/user/$USER/miniforge3/etc/profile.d/conda.sh


# check for tools available in the Ibex app stack
module avail toolX toolY toolZ
module load toolX toolY toolZ

# Check Nextflow and Singularity
module load nextflow singularity
nextflow -version
singularity --version

# check for tools installed using the conda env
conda activate metagenomics
gunc --version
...
conda deactivate
All commands should print a version number without errors. If a tool is missing from the conda environment, re-run the mamba create command from Step 3, or install the missing tool individually with mamba install -n metagenomics -c bioconda <toolname>.

Step 6: Download Course Dataset

All hands-on sessions use real sequencing data from a published synthetic mock community study (Pereira-Marques et al., 2019, BioProject PRJNA521492). Two samples are used:

  • mock_control (SRR9291172): 20-species mock community, 0% host DNA. Used throughout Modules 1–8.
  • mock_ss90 (SRR9291174): same community spiked with 90% C57BL/6 mouse host DNA. Used in Module 1 to demonstrate host read removal.
Host reference genome The host removal step (Module 1) uses the C57BL/6 mouse reference genome GRCm39 (NCBI: GCA_000001635.9). This genome is pre-staged by the instructors at /biocorelab/BIX/resources/genomes/mus_musculus/release-113/Mus_musculus.GRCm39.dna_sm.toplevel.fa. The exact path will be confirmed on Day 1.

Three tools from the SRA Toolkit are used:

ToolWhat it does
prefetchDownloads the compressed SRA container file to a local cache. Resumable if interrupted: safer than downloading FASTQ directly over a slow connection.
fasterq-dumpConverts the SRA file to FASTQ format. --split-files writes paired reads as separate R1 and R2 files; -p shows a progress bar; --outfile sets the output name prefix.
gzipCompresses FASTQ files. Almost all bioinformatics tools accept .fastq.gz directly and compression reduces file size by ~4×.
 
# Start an interactive compute session (download takes ~20–30 min)
srun --partition=batch --time=02:00:00 --mem=16G --cpus-per-task=8 --pty bash

# Navigate to your course working directory
cd /ibex/user/$USER/metagenomics_course

# Create directories for raw reads and a temporary SRA cache
mkdir -p raw_reads sra_cache
cd raw_reads

# Load SRA Toolkit (provides prefetch and fasterq-dump)
module load sratoolkit

# Step 1: Download SRA container files
# prefetch saves .sra files to sra_cache/: resumable if the connection drops
prefetch --verbose --progress --output-directory ../sra_cache SRR9291172
prefetch --verbose --progress --output-directory ../sra_cache SRR9291174

# Step 2: Convert SRA → paired FASTQ
# --split-files  writes mock_control_1.fastq and mock_control_2.fastq (R1 / R2)
# --outfile      sets the filename prefix (instead of using the SRR accession)
# -p / -v        show a progress bar and verbose output
# --temp         scratch space for intermediate files (cleaned up automatically)
fasterq-dump --split-files --threads 8 -p -v \
    --outfile mock_control \
    --temp ../sra_cache/tmp \
    ../sra_cache/SRR9291172/SRR9291172.sra

fasterq-dump --split-files --threads 8 -p -v \
    --outfile mock_ss90 \
    --temp ../sra_cache/tmp \
    ../sra_cache/SRR9291174/SRR9291174.sra

# Step 3: Compress (reduces size ~4×; required by most downstream tools)
gzip mock_control_1.fastq mock_control_2.fastq
gzip mock_ss90_1.fastq    mock_ss90_2.fastq

# Step 4: Remove the SRA cache: no longer needed
rm -rf ../sra_cache

# Verify: check the four output files and their sizes
ls -lh /ibex/user/$USER/metagenomics_course/raw_reads/

#########################################
## Let's peek inside these files
# print the top ten lines
zcat mock_ss90_2.fastq.gz | head
# Estimate number of sequences, bases etc?
# load the seqkit toolkit
module load seqkit;
seqkit stats -a -j 10 *.fastq.gz | tee fastq_stats.txt

# OR you can submit a non-interactive job to Slurm
sbatch -J seqkit_stats -c 20 -t 2:00:00 --mail-type=END \
--mail-user=myemail@kaust.edu.sa -e logs/slurm_seqkit_stats_%j.err -o stats_fastq.txt \
--wrap="module load seqkit; seqkit stats -a -j 10 *.fastq.gz"
Expected files after this step You should see four files in raw_reads/: mock_control_1.fastq.gz, mock_control_2.fastq.gz, mock_ss90_1.fastq.gz, mock_ss90_2.fastq.gz.

Step 6b: Download Long-Read Data (Zymo ONT)

The long-read modules use a Zymo ZymoBIOMICS mock community dataset sequenced on Oxford Nanopore GridION with R10 high-accuracy chemistry. Download it now into the same raw_reads/ directory. The file is large (~10 GB): submit it as a background job or run it inside your existing srun session.

cd /ibex/user/$USER/metagenomics_course/raw_reads

wget -q \
    https://s3.climb.ac.uk/nanopore/Zymo-GridION-EVEN-BB-SN-PCR-R10HC-flipflop.fq.gz \
    -O zymo-even_ont-r10_long.fastq.gz

# Verify the download
ls -lh zymo-even_ont-r10_long.fastq.gz

# count number sequences and fetch throughput
module load seqkit;
seqkit stats -a -j 10 zymo-even_ont-r10_long.fastq.gz | tee zymo-even_ont-r10_long.fastq.stats
Expected file raw_reads/zymo-even_ont-r10_long.fastq.gz: the Zymo R10HC ONT reads used in Module 1 (Steps 6–9) and Module 2 (long-read and hybrid assembly).
Want to learn more about SRA Toolkit and NCBI Datasets? The Linux & Ibex lab: Step 7 covers both tools in depth, including how to inspect SRA metadata before downloading and how to use the NCBI Datasets CLI to download reference genomes such as GRCm39.
