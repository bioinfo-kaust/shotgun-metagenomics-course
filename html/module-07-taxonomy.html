MODULE 7, DAY 4, 09:15 – 11:20

Taxonomy & Annotation

Classify MAGs phylogenomically with GTDB-Tk, annotate contigs with CAT, and functionally annotate bins with Prokka (prokaryotes) and MetaEuk (eukaryotes).

Learning Objectives

  • Classify MAGs using the Genome Taxonomy Database (GTDB) with GTDB-Tk
  • Understand differences between GTDB and NCBI taxonomy
  • Annotate contigs and bins at the sequence level with CAT
  • Functionally annotate prokaryotic bins with Prokka
  • Predict eukaryotic genes from bins with MetaEuk

Background

Taxonomy in metagenomics can be assigned at multiple levels. GTDB-Tk places MAGs into the GTDB reference tree using average nucleotide identity and phylogenomics; this is the gold standard for prokaryotic MAG classification. CAT assigns taxonomy to individual contigs or bins by searching predicted proteins against a database and applying a last-common-ancestor algorithm. Prokka rapidly annotates prokaryotic bins with gene functions, rRNA, tRNA, and CRISPRs. MetaEuk handles eukaryotic gene prediction in bins, which is important for complex environments (e.g. soil, marine) where eukaryotic microorganisms are abundant.

Hands-on

Step 1: Phylogenomic Classification with GTDB-Tk

cat > scripts/tax_01_gtdbtk.sh << 'EOF'
#!/bin/bash
#SBATCH --job-name=tax_01_gtdbtk
#SBATCH --partition=batch
#SBATCH --time=04:00:00
#SBATCH --mem=128G
#SBATCH --cpus-per-task=16
#SBATCH --output=logs/tax_01_gtdbtk_%j.out
#SBATCH --error=logs/tax_01_gtdbtk_%j.err
#SBATCH --mail-type=END,FAIL
#SBATCH --mail-user=

# Exit immediately on error (-e), treat unset variables as errors (-u),
# and propagate pipe failures (-o pipefail) so the job stops at the first
# sign of trouble rather than silently continuing with bad data.
set -eo pipefail
cd /ibex/user/$USER/metagenomics_course
mkdir -p results/07-taxonomy/gtdbtk

module load gtdb-tk

gtdbtk classify_wf \
    --genome_dir results/05-refinement/das_tool/mock_control_DASTool_bins/ \
    --out_dir results/07-taxonomy/gtdbtk \
    --extension fa \
    --cpus 16 \
    --mash_db /ibex/reference/KSL/gtdb-tk/release226/
EOF

Submit: sbatch scripts/tax_01_gtdbtk.sh  |  Monitor: squeue -u $USER

GTDB-Tk output Key files: gtdbtk.bac120.summary.tsv (bacteria) and gtdbtk.ar53.summary.tsv (archaea). Each row provides the classification, placement in the reference tree, RED value, and warnings. The classification column uses GTDB nomenclature (prefixed d__, p__, c__, etc.).
Verify against ground truth The mock_control sample contains reads from exactly 20 known bacterial species (ATCC MSA-1003 mock community). Cross-check the species recovered in your GTDB-Tk output against the known composition from Pereira-Marques et al. (2019). This is a rare opportunity in metagenomics to evaluate your full pipeline against a gold standard.

Step 2: Contig & Bin Taxonomy with CAT

cat > scripts/tax_02_cat.sh << 'EOF'
#!/bin/bash
#SBATCH --job-name=tax_02_cat
#SBATCH --partition=batch
#SBATCH --time=04:00:00
#SBATCH --mem=64G
#SBATCH --cpus-per-task=8
#SBATCH --output=logs/tax_02_cat_%j.out
#SBATCH --error=logs/tax_02_cat_%j.err
#SBATCH --mail-type=END,FAIL
#SBATCH --mail-user=

# Exit immediately on error (-e), treat unset variables as errors (-u),
# and propagate pipe failures (-o pipefail) so the job stops at the first
# sign of trouble rather than silently continuing with bad data.
set -eo pipefail
cd /ibex/user/$USER/metagenomics_course
mkdir -p results/07-taxonomy/cat

module load cat

CAT bins \
    -b results/05-refinement/das_tool/mock_control_DASTool_bins/ \
    -s .fa \
    -d /ibex/project/c2xxx/databases/cat_db/db \
    -t /ibex/project/c2xxx/databases/cat_db/taxonomy \
    -o results/07-taxonomy/cat/mock_control \
    --threads 8 \
    --index_chunks 1

CAT add_names \
    results/07-taxonomy/cat/mock_control.bin2classification.txt \
    -o results/07-taxonomy/cat/mock_control.named.txt \
    -t /ibex/project/c2xxx/databases/cat_db/taxonomy \
    --only_official
EOF

Submit: sbatch scripts/tax_02_cat.sh  |  Monitor: squeue -u $USER

Step 3: Prokaryotic Annotation with Prokka

cat > scripts/tax_03_prokka.sh << 'EOF'
#!/bin/bash
#SBATCH --job-name=tax_03_prokka
#SBATCH --partition=batch
#SBATCH --time=04:00:00
#SBATCH --mem=32G
#SBATCH --cpus-per-task=8
#SBATCH --output=logs/tax_03_prokka_%j.out
#SBATCH --error=logs/tax_03_prokka_%j.err
#SBATCH --mail-type=END,FAIL
#SBATCH --mail-user=

# Exit immediately on error (-e), treat unset variables as errors (-u),
# and propagate pipe failures (-o pipefail) so the job stops at the first
# sign of trouble rather than silently continuing with bad data.
set -eo pipefail
cd /ibex/user/$USER/metagenomics_course
mkdir -p results/07-taxonomy/prokka

module load prokka

for bin in results/05-refinement/das_tool/mock_control_DASTool_bins/*.fa; do
    name=$(basename "${bin%.fa}")
    prokka \
        "$bin" \
        --outdir "results/07-taxonomy/prokka/$name" \
        --prefix "$name" \
        --metagenome \
        --kingdom Bacteria \
        --cpus 8 \
        --force
done
EOF

Submit: sbatch scripts/tax_03_prokka.sh  |  Monitor: squeue -u $USER

Prokka output files .gff: gene coordinates (GFF3); .faa: protein sequences; .tsv: annotation table; .txt: summary statistics (# CDS, rRNA, tRNA).

Once Prokka finishes, count annotated CDS per bin interactively:

for f in results/07-taxonomy/prokka/*/; do
    txt=$(ls "$f"/*.txt 2>/dev/null | head -1)
    [ -f "$txt" ] && echo -e "$(basename $f)\t$(grep 'CDS:' $txt)"
done

Step 4: Eukaryotic Gene Prediction with MetaEuk

cat > scripts/tax_04_metaeuk.sh << 'EOF'
#!/bin/bash
#SBATCH --job-name=tax_04_metaeuk
#SBATCH --partition=batch
#SBATCH --time=04:00:00
#SBATCH --mem=64G
#SBATCH --cpus-per-task=8
#SBATCH --output=logs/tax_04_metaeuk_%j.out
#SBATCH --error=logs/tax_04_metaeuk_%j.err
#SBATCH --mail-type=END,FAIL
#SBATCH --mail-user=

# Exit immediately on error (-e), treat unset variables as errors (-u),
# and propagate pipe failures (-o pipefail) so the job stops at the first
# sign of trouble rather than silently continuing with bad data.
set -eo pipefail
cd /ibex/user/$USER/metagenomics_course
mkdir -p results/07-taxonomy/metaeuk

source $(conda info --base)/etc/profile.d/conda.sh 2>/dev/null ||\
    source /ibex/user/$USER/miniforge3/etc/profile.d/conda.sh
conda activate metagenomics

metaeuk easy-predict \
    results/05-refinement/das_tool/mock_control_DASTool_bins/bin.euk.fa \
    /ibex/project/c2xxx/databases/metaeuk_db/metaeuk_db \
    results/07-taxonomy/metaeuk/bin_euk \
    results/07-taxonomy/metaeuk/tmp \
    --threads 8
EOF

Submit: sbatch scripts/tax_04_metaeuk.sh  |  Monitor: squeue -u $USER

When to use MetaEuk MetaEuk is appropriate for bins that Tiara (Module 4) classified as eukarya. Running it on prokaryotic bins is wasteful. In nf-core/mag, MetaEuk is enabled with --skip_metaeuk false and requires a MetaEuk-compatible MMseqs2 database (--metaeuk_mmseqs_db).

Step 5: Compare GTDB vs CAT Taxonomy

Both outputs are available once their jobs complete: inspect them interactively:

# Show GTDB classifications for bacteria
column -t -s$'\t' \
    results/07-taxonomy/gtdbtk/gtdbtk.bac120.summary.tsv | \
    awk 'NR==1 || NR<=11' | cut -f1,2

# Compare with CAT classification
column -t -s$'\t' \
    results/07-taxonomy/cat/mock_control.named.txt | head -10
