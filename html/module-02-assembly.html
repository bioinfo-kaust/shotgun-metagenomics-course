MODULE 2, DAY 2, 09:15 – 14:30

Assembly, Evaluation & Annotation

Assemble cleaned reads into contigs, evaluate assembly quality, map reads for coverage, predict genes with Prodigal, and identify viral and plasmid sequences with geNomad.

Learning Objectives

  • Assemble short reads with MEGAHIT and metaSPAdes and compare results
  • Assemble long reads with Flye and metaMDBG
  • Perform hybrid assembly combining short and long reads with SPAdes
  • Evaluate assembly quality with QUAST and ALE
  • Map reads back to contigs with Bowtie2 (short) and Minimap2 (long) to generate coverage profiles
  • Predict protein-coding genes from contigs with Prodigal in metagenome mode
  • Identify viral sequences and plasmids with geNomad

Background

De novo assembly reconstructs genomes from overlapping reads without a reference. In metagenomics, each assembler uses different strategies: MEGAHIT uses succinct de Bruijn graphs optimised for memory efficiency; metaSPAdes uses iterative k-mer sizes and is better for complex communities; Flye uses repeat graphs suited to long reads. After assembly, coverage depth is obtained by re-mapping reads, essential for binning. Prodigal predicts ORFs at contig level (metagenome mode), providing gene coordinates that feed downstream annotation. geNomad identifies mobile genetic elements that should not be co-binned with chromosomal sequences.

Part A: Short Read Assembly

One-time setup Create the scripts/ and logs/ directories once at the start of this module. All batch scripts and their log files will live here throughout the course. 
cd /ibex/user/$USER/metagenomics_course
mkdir -p scripts logs

Step 1: Assemble with MEGAHIT

cat > scripts/asm_01_megahit.sh << 'EOF'
#!/bin/bash
#SBATCH --job-name=asm_01_megahit
#SBATCH --partition=batch
#SBATCH --time=02:00:00
#SBATCH --mem=64G
#SBATCH --cpus-per-task=16
#SBATCH --output=logs/asm_01_megahit_%j.out
#SBATCH --error=logs/asm_01_megahit_%j.err
#SBATCH --mail-type=END,FAIL
#SBATCH --mail-user=

# Exit immediately on error (-e), treat unset variables as errors (-u),
# and propagate pipe failures (-o pipefail) so the job stops at the first
# sign of trouble rather than silently continuing with bad data.
set -euo pipefail
cd /ibex/user/$USER/metagenomics_course
mkdir -p results/02-assembly/megahit

module load megahit

megahit \
    -1 results/01-qc/bbnorm/mock_control_1.norm.fastq.gz \
    -2 results/01-qc/bbnorm/mock_control_2.norm.fastq.gz \
    --out-dir results/02-assembly/megahit \
    --out-prefix mock_control \
    --min-contig-len 500 \
    -t 16
EOF

Submit: sbatch scripts/asm_01_megahit.sh  |  Monitor: squeue -u $USER

Part C: Assembly Evaluation

Step 6: Assembly Statistics with QUAST

QUAST runs in seconds on typical assemblies: run it interactively while you wait for assembler jobs to finish.

mkdir -p results/02-assembly/quast

module load quast

quast.py \

    --output-dir results/02-assembly/quast \
    --min-contig 500 \

Step 7: Assembly Likelihood Evaluation with ALE

ALE scores assemblies by mapping reads back and computing alignment likelihood: useful for selecting the best assembler output.

cat > scripts/asm_07_ale.sh << 'EOF'
#!/bin/bash
#SBATCH --job-name=asm_07_ale
#SBATCH --partition=batch
#SBATCH --time=02:00:00
#SBATCH --mem=32G
#SBATCH --cpus-per-task=8
#SBATCH --output=logs/asm_07_ale_%j.out
#SBATCH --error=logs/asm_07_ale_%j.err
#SBATCH --mail-type=END,FAIL
#SBATCH --mail-user=

# Exit immediately on error (-e), treat unset variables as errors (-u),
# and propagate pipe failures (-o pipefail) so the job stops at the first
# sign of trouble rather than silently continuing with bad data.
set -euo pipefail
cd /ibex/user/$USER/metagenomics_course
mkdir -p results/02-assembly/ale

module load bowtie2 samtools

source $(conda info --base)/etc/profile.d/conda.sh 2>/dev/null ||\
    source /ibex/user/$USER/miniforge3/etc/profile.d/conda.sh
conda activate metagenomics

bowtie2-build \
    results/02-assembly/megahit/mock_control.contigs.fa \
    results/02-assembly/ale/megahit_idx

bowtie2 \
    -x results/02-assembly/ale/megahit_idx \
    -1 results/01-qc/bbnorm/mock_control_1.norm.fastq.gz \
    -2 results/01-qc/bbnorm/mock_control_2.norm.fastq.gz \
    -p 8 | \
samtools sort -o results/02-assembly/ale/megahit.sorted.bam
samtools index results/02-assembly/ale/megahit.sorted.bam

ALE \
    results/02-assembly/ale/megahit.sorted.bam \
    results/02-assembly/megahit/mock_control.contigs.fa \
    results/02-assembly/ale/megahit.ale
EOF

Submit: sbatch scripts/asm_07_ale.sh  |  Monitor: squeue -u $USER

Part D: Coverage Mapping

Step 8: Map Short Reads for Coverage (Bowtie2)

The BAM produced here feeds directly into binning (Module 4): the coverage file tells binners how much each contig is represented across samples.

cat > scripts/asm_08_bowtie2_coverage.sh << 'EOF'
#!/bin/bash
#SBATCH --job-name=asm_08_bowtie2_coverage
#SBATCH --partition=batch
#SBATCH --time=01:00:00
#SBATCH --mem=16G
#SBATCH --cpus-per-task=8
#SBATCH --output=logs/asm_08_bowtie2_coverage_%j.out
#SBATCH --error=logs/asm_08_bowtie2_coverage_%j.err
#SBATCH --mail-type=END,FAIL
#SBATCH --mail-user=

# Exit immediately on error (-e), treat unset variables as errors (-u),
# and propagate pipe failures (-o pipefail) so the job stops at the first
# sign of trouble rather than silently continuing with bad data.
set -euo pipefail
cd /ibex/user/$USER/metagenomics_course
mkdir -p results/02-assembly/coverage

module load bowtie2 samtools

bowtie2-build \
    results/02-assembly/megahit/mock_control.contigs.fa \
    results/02-assembly/coverage/assembly_idx

bowtie2 \
    -x results/02-assembly/coverage/assembly_idx \
    -1 results/01-qc/bbnorm/mock_control_1.norm.fastq.gz \
    -2 results/01-qc/bbnorm/mock_control_2.norm.fastq.gz \
    -p 8 | \
samtools sort -o results/02-assembly/coverage/mock_control.sorted.bam
samtools index results/02-assembly/coverage/mock_control.sorted.bam

samtools coverage results/02-assembly/coverage/mock_control.sorted.bam \
    > results/02-assembly/coverage/mock_control.coverage.tsv
EOF

Submit: sbatch scripts/asm_08_bowtie2_coverage.sh  |  Monitor: squeue -u $USER

Part E: Contig Annotation

Step 10: Gene Prediction with Prodigal

Prodigal completes in under a minute for typical metagenome assemblies: run it interactively.

mkdir -p results/02-assembly/prodigal

module load prodigal

prodigal \
    -i results/02-assembly/megahit/mock_control.contigs.fa \
    -p meta \
    -a results/02-assembly/prodigal/predicted_proteins.faa \
    -d results/02-assembly/prodigal/predicted_genes.fna \
    -f gff \
    -o results/02-assembly/prodigal/predicted.gff
Always use -p meta The default -p single mode assumes a single genome and performs poorly on short, anonymous metagenomic contigs.
