MODULE 2 — Day 2, Morning

Assembly, Evaluation & Annotation

Assemble cleaned reads into contigs, evaluate assembly quality, map reads for coverage, predict genes with Prodigal, and identify viral and plasmid sequences with geNomad.

Learning Objectives

  • Assemble short reads with MEGAHIT and metaSPAdes and compare results
  • Assemble long reads with Flye and metaMDBG
  • Perform hybrid assembly combining short and long reads with SPAdes
  • Evaluate assembly quality with QUAST and ALE
  • Map reads back to contigs with Bowtie2 (short) and Minimap2 (long) to generate coverage profiles
  • Predict protein-coding genes from contigs with Prodigal in metagenome mode
  • Identify viral sequences and plasmids with geNomad

Background

De novo assembly reconstructs genomes from overlapping reads without a reference. In metagenomics, each assembler uses different strategies: MEGAHIT uses succinct de Bruijn graphs optimised for memory efficiency; metaSPAdes uses iterative k-mer sizes and is better for complex communities; Flye uses repeat graphs suited to long reads. After assembly, coverage depth is obtained by re-mapping reads — essential for binning. Prodigal predicts ORFs at contig level (metagenome mode), providing gene coordinates that feed downstream annotation. geNomad identifies mobile genetic elements that should not be co-binned with chromosomal sequences.

Part A — Short Read Assembly

Step 1 — Assemble with MEGAHIT

mkdir -p results/02-assembly/megahit

megahit \
    -1 results/01-qc/bbnorm/sample_R1.norm.fastq.gz \
    -2 results/01-qc/bbnorm/sample_R2.norm.fastq.gz \
    --out-dir results/02-assembly/megahit \
    --out-prefix sample \
    --min-contig-len 500 \
    -t 16

Step 2 — Assemble with metaSPAdes

mkdir -p results/02-assembly/metaspades

spades.py \
    --meta \
    -1 results/01-qc/bbnorm/sample_R1.norm.fastq.gz \
    -2 results/01-qc/bbnorm/sample_R2.norm.fastq.gz \
    -o results/02-assembly/metaspades \
    --threads 16 \
    --memory 128

Step 3 — Hybrid Assembly with SPAdes

mkdir -p results/02-assembly/hybrid

spades.py \
    --meta \
    -1 results/01-qc/bbnorm/sample_R1.norm.fastq.gz \
    -2 results/01-qc/bbnorm/sample_R2.norm.fastq.gz \
    --nanopore results/01-qc/long_host_removed/sample.decontam.fastq.gz \
    -o results/02-assembly/hybrid \
    --threads 16 \
    --memory 128

Part B — Long Read Assembly

Step 4 — Assemble with Flye

mkdir -p results/02-assembly/flye

flye \
    --nano-raw results/01-qc/long_host_removed/sample.decontam.fastq.gz \
    --out-dir results/02-assembly/flye \
    --meta \
    --threads 16
Flye read type flags Use --nano-raw for older Guppy basecalling, --nano-hq for high-accuracy basecalled reads (Q20+), and --pacbio-raw / --pacbio-hifi for PacBio data.

Step 5 — Assemble with metaMDBG

mkdir -p results/02-assembly/metamdbg

metaMDBG asm \
    --out-dir results/02-assembly/metamdbg \
    --in-ont results/01-qc/long_host_removed/sample.decontam.fastq.gz \
    --threads 16

Part C — Assembly Evaluation

Step 6 — Assembly Statistics with QUAST

mkdir -p results/02-assembly/quast

quast.py \
    results/02-assembly/megahit/sample.contigs.fa \
    results/02-assembly/metaspades/scaffolds.fasta \
    --output-dir results/02-assembly/quast \
    --min-contig 500 \
    --threads 8 \
    --labels "MEGAHIT,metaSPAdes"

Step 7 — Assembly Likelihood Evaluation with ALE

ALE scores assemblies by mapping reads back and computing alignment likelihood — useful for selecting the best assembler output.

mkdir -p results/02-assembly/ale

# Map reads to assembly
bowtie2-build results/02-assembly/megahit/sample.contigs.fa \
    results/02-assembly/ale/megahit_idx

bowtie2 \
    -x results/02-assembly/ale/megahit_idx \
    -1 results/01-qc/bbnorm/sample_R1.norm.fastq.gz \
    -2 results/01-qc/bbnorm/sample_R2.norm.fastq.gz \
    -p 8 | \
samtools sort -o results/02-assembly/ale/megahit.sorted.bam
samtools index results/02-assembly/ale/megahit.sorted.bam

# Run ALE
ALE \
    results/02-assembly/ale/megahit.sorted.bam \
    results/02-assembly/megahit/sample.contigs.fa \
    results/02-assembly/ale/megahit.ale

Part D — Coverage Mapping

Step 8 — Map Short Reads for Coverage (Bowtie2)

mkdir -p results/02-assembly/coverage

# Build index on chosen assembly
bowtie2-build \
    results/02-assembly/megahit/sample.contigs.fa \
    results/02-assembly/coverage/assembly_idx

# Align reads
bowtie2 \
    -x results/02-assembly/coverage/assembly_idx \
    -1 results/01-qc/bbnorm/sample_R1.norm.fastq.gz \
    -2 results/01-qc/bbnorm/sample_R2.norm.fastq.gz \
    -p 8 | \
samtools sort -o results/02-assembly/coverage/sample.sorted.bam
samtools index results/02-assembly/coverage/sample.sorted.bam

# Compute per-contig coverage depth
samtools coverage results/02-assembly/coverage/sample.sorted.bam \
    > results/02-assembly/coverage/sample.coverage.tsv

Step 9 — Map Long Reads for Coverage (Minimap2)

minimap2 \
    -ax map-ont \
    results/02-assembly/flye/assembly.fasta \
    results/01-qc/long_host_removed/sample.decontam.fastq.gz \
    -t 8 | \
samtools sort -o results/02-assembly/coverage/sample_long.sorted.bam
samtools index results/02-assembly/coverage/sample_long.sorted.bam

Part E — Contig Annotation

Step 10 — Gene Prediction with Prodigal

mkdir -p results/02-assembly/prodigal

prodigal \
    -i results/02-assembly/megahit/sample.contigs.fa \
    -p meta \
    -a results/02-assembly/prodigal/predicted_proteins.faa \
    -d results/02-assembly/prodigal/predicted_genes.fna \
    -f gff \
    -o results/02-assembly/prodigal/predicted.gff
Always use -p meta The default -p single mode assumes a single genome and performs poorly on short, anonymous metagenomic contigs.

Step 11 — Virus & Plasmid Detection with geNomad

mkdir -p results/02-assembly/genomad

genomad end-to-end \
    results/02-assembly/megahit/sample.contigs.fa \
    results/02-assembly/genomad \
    /path/to/genomad_db \
    --threads 8 \
    --cleanup
geNomad output Key files: *_summary/*_virus_summary.tsv and *_summary/*_plasmid_summary.tsv. Contigs identified as viral or plasmid origin should be excluded from binning. In nf-core/mag, enable with --run_virus_identification true --genomad_db /path/to/genomad_db.

Key nf-core/mag Parameters

ParameterDefaultDescription
--assemblermegahitAssembler: megahit, spades
--spades_options—Extra SPAdes flags
--megahit_options—Extra MEGAHIT flags
--skip_quastfalseSkip QUAST assembly evaluation
--skip_prodigalfalseSkip Prodigal gene prediction
--run_virus_identificationfalseRun geNomad virus/plasmid detection
--genomad_db—Path to geNomad database
--longreads_use_illumina_backbonefalseUse hybrid SPAdes assembly with Illumina backbone
