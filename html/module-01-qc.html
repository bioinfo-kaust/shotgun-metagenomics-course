MODULE 1, Day 1

QC & Preprocessing

Adapter removal, host and PhiX decontamination, quality filtering, and coverage normalisation for short reads; adapter removal, quality filtering, and host decontamination for long reads.

Learning Objectives

  • Assess raw read quality with FastQC and interpret key metrics
  • Trim adapters and apply quality filters with fastp
  • Remove host-derived reads by aligning to a reference genome with Bowtie2
  • Remove PhiX spike-in reads using a Bowtie2 PhiX index
  • Normalise coverage depth with BBNorm to improve assembly
  • Assess long read quality with NanoPlot
  • Remove long read adapters with Porechop_ABI
  • Filter long reads by quality and length with Chopper, Filtlong, or Nanoq

Background

High-quality input reads are critical for all downstream metagenomics steps. Short read preprocessing removes adapter sequences, trims low-quality bases, and eliminates reads originating from the host organism or the PhiX sequencing control. Coverage normalisation with BBNorm reduces extreme depth variation, which improves de novo assembly of complex communities. For long reads (Oxford Nanopore or PacBio), different tools handle adapter ligation artefacts and basecalling quality thresholds before host decontamination.

Course dataset This module uses reads from a published synthetic mock community study (Pereira-Marques et al., 2019, BioProject PRJNA521492). mock_control (SRR9291172) contains reads from 20 known bacterial species with no host contamination; you can verify your taxonomy results against the known ground truth in Module 7. A second sample, mock_ss90 (SRR9291174), contains the same community spiked with 90% mouse host DNA, and is introduced in Step 3 to demonstrate why host read removal matters.

Step 3: Host Read Removal with Bowtie2

Why host removal matters: mock_ss90 demonstration The mock_ss90 sample (SRR9291174) was deliberately spiked with 90% mouse (C57BL/6) host DNA. Running host removal on it removes ~90% of reads, leaving only the microbial fraction. The script below runs on both mock_control (0% host) and mock_ss90 (90% host); compare the log output to see the dramatic difference in the number of reads discarded. 
cat > scripts/qc_03_host_removal.sh << 'EOF'
#!/bin/bash
#SBATCH --job-name=qc_03_host
#SBATCH --partition=batch
#SBATCH --time=04:00:00
#SBATCH --mem=32G
#SBATCH --cpus-per-task=8
#SBATCH --output=logs/qc_03_host_%j.out
#SBATCH --error=logs/qc_03_host_%j.err
#SBATCH --mail-type=END,FAIL
#SBATCH --mail-user=

# Exit immediately on error (-e), treat unset variables as errors (-u),
# and propagate pipe failures (-o pipefail) so the job stops at the first
# sign of trouble rather than silently continuing with bad data.
set -euo pipefail
cd /ibex/user/$USER/metagenomics_course
mkdir -p results/01-qc/host_removal db/host_index

# Reference genome: C57BL/6 mouse GRCm39 (pre-staged by instructors)
HOST_REF=/biocorelab/BIX/resources/genomes/mus_musculus/release-113/Mus_musculus.GRCm39.dna_sm.toplevel.fa

module load bowtie2 samtools

# Build index once; skipped if already present
if [ ! -f db/host_index/GRCm39.1.bt2 ]; then
    bowtie2-build $HOST_REF db/host_index/GRCm39
fi

# mock_control (0% host)
bowtie2 \
    -x db/host_index/GRCm39 \
    -1 results/01-qc/fastp/mock_control_1.trimmed.fastq.gz \
    -2 results/01-qc/fastp/mock_control_2.trimmed.fastq.gz \
    --un-conc-gz results/01-qc/host_removal/mock_control_%.host_removed.fastq.gz \
    -p 8 --sensitive \
    -S /dev/null 2> results/01-qc/host_removal/mock_control.host.log

# mock_ss90 (90% host); compare to see effect of contamination
bowtie2 \
    -x db/host_index/GRCm39 \
    -1 raw_reads/mock_ss90_1.fastq.gz \
    -2 raw_reads/mock_ss90_2.fastq.gz \
    --un-conc-gz results/01-qc/host_removal/mock_ss90_%.host_removed.fastq.gz \
    -p 8 --sensitive \
    -S /dev/null 2> results/01-qc/host_removal/mock_ss90.host.log

grep "overall alignment rate" results/01-qc/host_removal/*.host.log
EOF

Submit: sbatch scripts/qc_03_host_removal.sh  |  Monitor: squeue -u $USER

bowtie2 note bowtie2 --un-conc-gz     Write paired-end reads that fail to align concordantly to file(s) at <path>. These reads correspond to the SAM records with the FLAGS 0x4 bit set and either the 0x40 or 0x80 bit set (depending on whether it's mate #1 or #2).

SAM flags 4 (0x4): The segment is unmapped.
64 (0x40): This is the first read in the pair.
128 (0x80): This is the last read in the pair.
nf-core/mag Provide the host genome with --host_fasta /biocorelab/BIX/resources/genomes/mus_musculus/release-113/Mus_musculus.GRCm39.dna_sm.toplevel.fa. The pipeline builds the Bowtie2 index automatically and reports host removal statistics per sample.

Step 4: PhiX Spike-in Removal with Bowtie2

cat > scripts/qc_04_phix_removal.sh << 'EOF'
#!/bin/bash
#SBATCH --job-name=qc_04_phix
#SBATCH --partition=batch
#SBATCH --time=01:00:00
#SBATCH --mem=8G
#SBATCH --cpus-per-task=8
#SBATCH --output=logs/qc_04_phix_%j.out
#SBATCH --error=logs/qc_04_phix_%j.err
#SBATCH --mail-type=END,FAIL
#SBATCH --mail-user=

# Exit immediately on error (-e), treat unset variables as errors (-u),
# and propagate pipe failures (-o pipefail) so the job stops at the first
# sign of trouble rather than silently continuing with bad data.
set -euo pipefail
cd /ibex/user/$USER/metagenomics_course

PHIX_URL="ftp://ftp.ncbi.nlm.nih.gov/genomes/genbank/viral/Enterobacteria_phage_phiX174_sensu_lato/all_assembly_versions/GCA_002596845.1_ASM259684v1/GCA_002596845.1_ASM259684v1_genomic.fna.gz"
if [ ! -f references/phix174.fna.gz ]; then
    mkdir -p references db/phix_index
    wget -q "$PHIX_URL" -O references/phix174.fna.gz
fi

module load bowtie2

# Build PhiX index once; skipped if already present
if [ ! -f db/phix_index/phix.1.bt2 ]; then
    bowtie2-build references/phix174.fna.gz db/phix_index/phix
fi

bowtie2 \
    -x db/phix_index/phix \
    -1 results/01-qc/host_removal/mock_control_1.host_removed.fastq.gz \
    -2 results/01-qc/host_removal/mock_control_2.host_removed.fastq.gz \
    --un-conc-gz results/01-qc/host_removal/mock_control_%.phix_removed.fastq.gz \
    -p 8 --sensitive \
    -S /dev/null 2> results/01-qc/host_removal/mock_control.phix.log
EOF

Submit: sbatch scripts/qc_05_bbnorm.sh  |  Monitor: squeue -u $USER

When to normalise BBNorm is most beneficial for highly complex communities (soil, sediment) with extreme coverage variation. For samples with more even coverage (e.g. gut), it may not be necessary and can reduce rare-taxon representation. Enable with --bbnorm true in nf-core/mag.

Step 9: Long Read Host Removal with Minimap2

cat > scripts/qc_09_longhost_removal.sh << 'EOF'
#!/bin/bash
#SBATCH --job-name=qc_09_longhost
#SBATCH --partition=batch
#SBATCH --time=02:00:00
#SBATCH --mem=16G
#SBATCH --cpus-per-task=8
#SBATCH --output=logs/qc_09_longhost_%j.out
#SBATCH --error=logs/qc_09_longhost_%j.err
#SBATCH --mail-type=END,FAIL
#SBATCH --mail-user=

# Exit immediately on error (-e), treat unset variables as errors (-u),
# and propagate pipe failures (-o pipefail) so the job stops at the first
# sign of trouble rather than silently continuing with bad data.
set -euo pipefail
cd /ibex/user/$USER/metagenomics_course
mkdir -p results/01-qc/long_host_removed

# Host reference (same GRCm39 as used for short reads).
# Zymo is a pure microbial mock community; alignment rate will be ~0%,
# which is the expected result and demonstrates the workflow.
HOST_REF=/biocorelab/BIX/resources/genomes/mus_musculus/release-113/Mus_musculus.GRCm39.dna_sm.toplevel.fa

module load minimap2 samtools
minimap2 \
    -ax map-ont \
    $HOST_REF \
    results/01-qc/filtered/zymo_long.chopper.fastq.gz \
    -t 8 | \
samtools view -b -f 4 | \
samtools fastq - | \
gzip > results/01-qc/long_host_removed/zymo_long.decontam.fastq.gz
EOF

Submit: sbatch scripts/qc_09_longhost_removal.sh  |  Monitor: squeue -u $USER
