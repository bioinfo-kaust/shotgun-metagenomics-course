MODULE 8, DAY 4, 11:20 – 12:00

Run Summary

Aggregate QC and tool metrics across all samples with MultiQC, generate per-sample MAG recovery tables, and explore the complete MAG catalogue with BigMAG. Then run the full nf-core/mag pipeline end-to-end.

Learning Objectives

  • Aggregate per-sample QC and tool output metrics into a single MultiQC report
  • Generate a structured bin summary table across all samples
  • Explore recovered MAG completeness, taxonomy, and quality visually with BigMAG
  • Run the complete nf-core/mag pipeline end-to-end from a samplesheet
  • Compare automated pipeline output to manual results from earlier modules

Background

After running all individual modules, the final step is to aggregate results into summary reports. MultiQC scans output directories for logs from FastQC, fastp, QUAST, Bowtie2, and dozens of other tools and assembles them into a single interactive HTML report. A bin summary table collects completeness, contamination, taxonomy, and quality tier for every MAG across all samples. BigMAG provides an interactive visualisation of the recovered MAG catalogue for exploring genome size, quality, and taxonomic diversity. The nf-core/mag pipeline automates all steps from Modules 1–8 with a single nextflow run command.

Step 1: Aggregate Reports with MultiQC

# Navigate to your course working directory
cd /ibex/user/$USER/metagenomics_course

module load multiqc

mkdir -p results/08-summary/multiqc

multiqc \
    results/ \
    --outdir results/08-summary/multiqc \
    --filename metagenomics_report \
    --title "Shotgun Metagenomics QC Report" \
    --comment "nf-core/mag v5.4.2 run summary"
MultiQC modules included MultiQC automatically detects and includes: FastQC, fastp, Bowtie2 alignment stats, QUAST assembly metrics, Prokka annotation counts, and CheckM2/BUSCO completeness: giving a full picture of every sample in one place.

Step 2: Generate Bin Summary Table

mkdir -p results/08-summary/bin_summary

# Combine CheckM2 quality report with GTDB-Tk taxonomy
join -t$'\t' -1 1 -2 1 \
    <(sort results/06-bin-qc/checkm2/quality_report.tsv) \
    <(sort results/07-taxonomy/gtdbtk/gtdbtk.bac120.summary.tsv | \
      cut -f1,2) \
    > results/08-summary/bin_summary/combined_summary.tsv

# Add MIMAG quality tier
awk -F'\t' 'NR==1 {print $0"\tMIMAG"; next}
{
    if ($2 >= 90 && $3 <= 5) q="HQ"
    else if ($2 >= 50 && $3 < 10) q="MQ"
    else q="LQ"
    print $0"\t"q
}' results/08-summary/bin_summary/combined_summary.tsv \
    > results/08-summary/bin_summary/final_summary.tsv

# Print summary counts
echo "=== MAG Recovery Summary ==="
echo "Total bins: $(tail -n +2 results/08-summary/bin_summary/final_summary.tsv | wc -l)"
echo "High-quality: $(awk -F'\t' '$NF=="HQ"' results/08-summary/bin_summary/final_summary.tsv | wc -l)"
echo "Medium-quality: $(awk -F'\t' '$NF=="MQ"' results/08-summary/bin_summary/final_summary.tsv | wc -l)"

Step 3: Visualise with BigMAG

mkdir -p results/08-summary/bigmag

source $(conda info --base)/etc/profile.d/conda.sh 2>/dev/null ||\
    source /ibex/user/$USER/miniforge3/etc/profile.d/conda.sh
conda activate metagenomics

bigmag \
    --checkm results/06-bin-qc/checkm2/quality_report.tsv \
    --gtdbtk results/07-taxonomy/gtdbtk/gtdbtk.bac120.summary.tsv \
    --output results/08-summary/bigmag \
    --title "Sample MAG Catalogue"

# Open the interactive report
open results/08-summary/bigmag/bigmag_report.html

Step 4: Full nf-core/mag Pipeline Run

Run the complete automated pipeline from a samplesheet in one command:

# Example samplesheet (CSV format)
cat samplesheet.csv
sample,group,short_reads_1,short_reads_2,long_reads
mock_control,group1,raw_reads/mock_control_1.fastq.gz,raw_reads/mock_control_2.fastq.gz,
mock_ss90,group1,raw_reads/mock_ss90_1.fastq.gz,raw_reads/mock_ss90_2.fastq.gz,
nextflow run nf-core/mag \
    -r 5.4.2 \
    -profile kaust \
    --input samplesheet.csv \
    --outdir results/nfcore_mag \
    --host_fasta data/reference/host_genome.fa \
    --bbnorm true \
    --assembler megahit \
    --skip_comebin false \
    --skip_semibin false \
    --run_virus_identification true \
    --genomad_db /path/to/genomad_db \
    --skip_gunc false \
    --gunc_db /path/to/gunc_db/gunc_db_progenomes2.1.dmnd \
    --checkm2_db /path/to/checkm2_db/uniref100.KO.1.dmnd \
    --gtdb_db /ibex/reference/KSL/gtdb-tk/release226/ \
    --max_cpus 32 \
    --max_memory 256.GB
