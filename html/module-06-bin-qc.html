MODULE 6, DAY 3, 14:00 – 15:30

Bin Quality Evaluation

Assess completeness, contamination, and chimeric content of MAGs using CheckM2, BUSCO, GUNC, and QUAST. Apply MIMAG quality thresholds to classify bins.

Learning Objectives

  • Estimate MAG completeness and contamination with CheckM2 (or CheckM)
  • Assess gene-set completeness with BUSCO against lineage databases
  • Detect chimeric bins with GUNC using taxonomic concordance scoring
  • Evaluate assembly-level statistics per bin with QUAST
  • Apply MIMAG standards to classify bins as high-, medium-, or low-quality

Background

MAG quality is assessed primarily by completeness (how much of the expected genome is present) and contamination (how much foreign sequence is included). CheckM2 uses a machine-learning model and is more accurate than the original CheckM, particularly for novel lineages without well-represented marker genes. BUSCO takes a complementary approach, searching for a lineage-specific set of near-universal single-copy orthologs. GUNC detects chimeric bins, where contigs from multiple species have been co-binned, using a taxonomic concordance score. QUAST reports contig-level assembly statistics per bin.

The MIMAG standards define: high-quality (≥90% completeness, ≤5% contamination, presence of 5S/16S/23S rRNA + ≥18 tRNA genes); medium-quality (≥50% completeness, <10% contamination); low-quality (below medium).

Hands-on

Step 1: Completeness & Contamination with CheckM2

cat > scripts/binqc_01_checkm2.sh << 'EOF'
#!/bin/bash
#SBATCH --job-name=binqc_01_checkm2
#SBATCH --partition=batch
#SBATCH --time=02:00:00
#SBATCH --mem=32G
#SBATCH --cpus-per-task=8
#SBATCH --output=logs/binqc_01_checkm2_%j.out
#SBATCH --error=logs/binqc_01_checkm2_%j.err
#SBATCH --mail-type=END,FAIL
#SBATCH --mail-user=

# Exit immediately on error (-e), treat unset variables as errors (-u),
# and propagate pipe failures (-o pipefail) so the job stops at the first
# sign of trouble rather than silently continuing with bad data.
set -eo pipefail
cd /ibex/user/$USER/metagenomics_course
mkdir -p results/06-bin-qc/checkm2

module load checkm2

checkm2 predict \
    --input results/05-refinement/das_tool/mock_control_DASTool_bins/ \
    --output-directory results/06-bin-qc/checkm2 \
    --database_path /biocorelab/BIX/resources/databases/CheckM2_database/uniref100.KO.1.dmnd \
    --extension fa \
    --threads 8
EOF

Submit: sbatch scripts/binqc_01_checkm2.sh  |  Monitor: squeue -u $USER

Once complete, view the completeness and contamination results interactively:

column -t -s$'\t' results/06-bin-qc/checkm2/quality_report.tsv | head -20
CheckM2 vs CheckM CheckM2 uses a machine-learning model trained on a broad genome database and is more accurate for novel lineages. CheckM remains an option with --skip_checkm false in nf-core/mag.

Step 3: Gene Set Completeness with BUSCO

cat > scripts/binqc_03_busco.sh << 'EOF'
#!/bin/bash
#SBATCH --job-name=binqc_03_busco
#SBATCH --partition=batch
#SBATCH --time=04:00:00
#SBATCH --mem=32G
#SBATCH --cpus-per-task=8
#SBATCH --output=logs/binqc_03_busco_%j.out
#SBATCH --error=logs/binqc_03_busco_%j.err
#SBATCH --mail-type=END,FAIL
#SBATCH --mail-user=

# Exit immediately on error (-e), treat unset variables as errors (-u),
# and propagate pipe failures (-o pipefail) so the job stops at the first
# sign of trouble rather than silently continuing with bad data.
set -eo pipefail
cd /ibex/user/$USER/metagenomics_course
mkdir -p results/06-bin-qc/busco

module load busco

for bin in results/05-refinement/das_tool/mock_control_DASTool_bins/*.fa; do
    name=$(basename "${bin%.fa}")
    busco \
        -i "$bin" \
        -o "results/06-bin-qc/busco/$name" \
        -m genome \
        -l bacteria_odb10 \
        -c 8
done
EOF

Submit: sbatch scripts/binqc_03_busco.sh  |  Monitor: squeue -u $USER

Step 4: Chimera Detection with GUNC

cat > scripts/binqc_04_gunc.sh << 'EOF'
#!/bin/bash
#SBATCH --job-name=binqc_04_gunc
#SBATCH --partition=batch
#SBATCH --time=02:00:00
#SBATCH --mem=32G
#SBATCH --cpus-per-task=8
#SBATCH --output=logs/binqc_04_gunc_%j.out
#SBATCH --error=logs/binqc_04_gunc_%j.err
#SBATCH --mail-type=END,FAIL
#SBATCH --mail-user=

# Exit immediately on error (-e), treat unset variables as errors (-u),
# and propagate pipe failures (-o pipefail) so the job stops at the first
# sign of trouble rather than silently continuing with bad data.
set -eo pipefail
cd /ibex/user/$USER/metagenomics_course
mkdir -p results/06-bin-qc/gunc

source $(conda info --base)/etc/profile.d/conda.sh 2>/dev/null ||\
    source /ibex/user/$USER/miniforge3/etc/profile.d/conda.sh

    --threads 8 \
    --file_suffix .fa

Step 6: Apply MIMAG Quality Thresholds

awk -F'\t' 'NR==1 {print $0"\tMIMAG_quality"; next}
{
    if ($2 >= 90 && $3 <= 5) quality = "High-quality"
    else if ($2 >= 50 && $3 < 10) quality = "Medium-quality"
    else quality = "Low-quality"
    print $0"\t"quality
}' results/06-bin-qc/checkm2/quality_report.tsv \
    > results/06-bin-qc/bins_mimag_classified.tsv

awk -F'\t' 'NR>1 {counts[$NF]++} END {for (q in counts) print q": "counts[q]}' \
    results/06-bin-qc/bins_mimag_classified.tsv
